# Polymarket BTC 5-Minute Market Maker Bot

Автоматизированный маркет-мейкер для торговли на [Polymarket](https://polymarket.com) в бинарных рынках **BTC Up/Down** с окнами по 5 минут.

## Как это работает

Polymarket каждые 5 минут создаёт рынок вида:

> **«Вырастет ли BTC к концу этого 5-минутного окна?»**

Рынок разрешается оракулом Chainlink: YES-токен стоит $1 если BTC закрылся выше цены открытия, иначе $0. NO-токен — наоборот.

Бот:
1. Подключается к Binance WebSocket и получает live-цену BTC в реальном времени
2. Определяет текущий рынок через Polymarket Gamma API по детерминированному slug-у (`btc-updown-5m-{timestamp}`)
3. За **последние 10 секунд** каждого окна вычисляет вероятность роста/падения по логистической функции
4. При сильном сигнале (>94% уверенности) выставляет maker-ордер на покупку YES или NO токена по цене **0.92 USDC**
5. Обновляет котировки каждые 200 мс через cancel/replace
6. За 2 секунды до закрытия снимает все ордера (защита от постфактум-заполнения)

## Стратегия и математика

### Сигнал

Бот сравнивает текущую цену BTC с ценой открытия 5-минутного окна и вычисляет вероятность роста через **логистическую функцию**:

```
p_up = 1 / (1 + exp(-k × ret))
ret  = (mid_сейчас - open_цена_окна) / open_цена_окна
k    = 2000  (калибровочный коэффициент)
```

### Калибровка коэффициента k

```
σ₅           = 0.22%   (волатильность BTC за 5 мин при σ_годовой = 60%)
σ_остаток    = σ₅ × √(10с / 300с) = 0.040%   (неопределённость за последние 10с)
k_точный     = 4421    (точная калибровка при σ₅ = 0.22%)
k_безопасный = 2000    (используется: сохраняет EV > 0 при σ₅ до 0.50%)
```

### Ожидаемые результаты (при настройках по умолчанию)

| Условие | Теор. win rate | EV / сделку |
|---|---|---|
| Обычный день (σ₅ = 0.22%) | **99.97%** | **+4.33 USDC** |
| Высокая волатильность (σ₅ = 0.50%) | 93.41% | +0.77 USDC |
| Экстремальная (σ₅ > 0.70%) | — | заблокировано |

- **Безубыточный win rate:** 92.0% (= TARGET_PRICE_YES)
- **Частота входов:** ~52.5% окон ≈ **151 сделка/день**
- **Потенциальный EV:** ~650 USDC/день (теоретически, при 50 USDC/сделке)

### Volatility Gate

Если high-low диапазон последней 5-минутной свечи Binance превышает **200 bps (2%)** — бот не торгует в этом окне. Это защита от флэш-крэшей и новостных движений, где модель случайного блуждания ломается.

### Заработок маркет-мейкера

Polymarket выплачивает **USDC-ребейты** мейкерам из тейкерских комиссий ежедневно. Основной источник прибыли — захват спреда: покупка токена по 0.92 при справедливой стоимости 0.94–0.99.

## Архитектура

```
main.py
├── ws_orderbook.py        Binance WebSocket (BTC/USDT orderbook + 5m candles)
├── market_calculator.py   Gamma API client + логистический сигнал
├── market_maker.py        Основной trading loop (tick → quote → cancel/replace)
├── polymarket_client.py   CLOB REST client (EIP-712 подпись, ордера, отмена)
├── stats.py               Трекер win rate и P&L статистики
└── config.py              Параметры из .env
```

### Поток данных

```
Binance WS                 Gamma API (каждые 10 мин)
    │                           │
    ▼                           ▼
OrderBookWS            MarketCalculator
(mid_price, candle)    (текущий рынок, token IDs)
         │                  │
         └──────┬───────────┘
                ▼
           MarketMaker (_tick каждые 200мс)
                │
         ┌──────┴───────┐
    _quote_window()   _rollover()
         │                 │
    PolymarketClient   BotStats.record_trade()
    (EIP-712 sign)
    POST /order
```

## Установка и настройка

### 1. Клонирование и установка зависимостей

```bash
git clone <repo-url>
cd <repo-dir>
pip install -r requirements.txt
```

### 2. Настройка файла `.env`

**Рекомендуемый способ — интерактивный помощник:**

```bash
python3 setup.py
```

Скрипт проведёт через все параметры на русском языке с подсказками и валидацией.

**Или вручную:**

```bash
cp .env.example .env
```

Откройте `.env` и укажите **приватный ключ** вашего кошелька:

```env
POLYMARKET_PRIVATE_KEY=0x<ваш_приватный_ключ>
```

> **Безопасность:** Никогда не храните приватный ключ в репозитории. Файл `.env` находится в `.gitignore`.

### 3. Одобрение контрактов Polymarket (разово)

Перед первым запуском необходимо дать approve на:
- `USDC.e` → контракт Exchange (`0x4bFb41d5B3570DeFd03C39a9A4D8dE6Bd8B8982E`)
- Conditional Tokens ERC-1155 → контракт Exchange

Сделать это можно через [app.polymarket.com](https://app.polymarket.com) — войдите кошельком и сделайте любую ставку, или используйте скрипт одобрения из SDK Polymarket.

Бот при старте проверяет наличие апрувов через `GET /auth/approvals` и выводит ошибку если они не установлены.

### 4. Пополнение баланса

Пополните кошелёк **USDC.e на сети Polygon**:
- Через Polymarket → Deposit
- Через bridge (Polygon Bridge, Stargate и др.)

Минимальный рекомендуемый баланс: `ORDER_SIZE_USDC × 10` (500 USDC по умолчанию).

## Запуск

### Обычный запуск

```bash
python main.py
```

### С уровнем логирования DEBUG

```bash
LOG_LEVEL=DEBUG python main.py
```

### Остановка

Нажмите `Ctrl+C` — бот корректно завершит работу:
1. Отменит все открытые ордера
2. Выведет итоговую статистику сессии
3. Закроет WebSocket соединение

### Пример стартового лога

```
2026-03-20T10:00:01.123 [INFO] main – BTC 5m market maker starting | wallet=0x1234...abcd
2026-03-20T10:00:01.456 [INFO] main – BTC mid-price: 87143.21
2026-03-20T10:00:01.891 [INFO] market_calculator – Fetched 6 BTC 5m markets from Gamma
2026-03-20T10:00:01.892 [INFO] market_maker – MarketMaker starting
2026-03-20T10:00:01.893 [INFO] market_maker – New window: 1742478300 → 1742478600
```

### Запуск в фоне на сервере (systemd)

Создайте `/etc/systemd/system/btc-mm.service`:

```ini
[Unit]
Description=Polymarket BTC Market Maker
After=network-online.target

[Service]
WorkingDirectory=/path/to/bot
ExecStart=/usr/bin/python3 main.py
EnvironmentFile=/path/to/bot/.env
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable btc-mm
sudo systemctl start btc-mm
sudo journalctl -u btc-mm -f
```

## Параметры конфигурации

Все параметры задаются через `.env` (или переменные окружения).

### Обязательные

| Параметр | Описание |
|---|---|
| `POLYMARKET_PRIVATE_KEY` | Приватный ключ EOA-кошелька (hex, с `0x`) |

### Торговые параметры

| Параметр | По умолчанию | Описание |
|---|---|---|
| `ORDER_SIZE_USDC` | `50` | Размер каждого ордера в USDC |
| `TARGET_PRICE_YES` | `0.92` | Цена котировки YES-токена (92 цента) |
| `TARGET_PRICE_NO` | `0.92` | Цена котировки NO-токена (92 цента) |
| `MIN_EDGE_BPS` | `50` | Минимальный edge в bps для входа (50 = 0.5%) |
| `MAX_EXPOSURE_USDC` | `500` | Максимальная суммарная экспозиция в USDC |
| `MAX_OPEN_ORDERS` | `4` | Макс. одновременных открытых ордеров |

### Тайминг

| Параметр | По умолчанию | Описание |
|---|---|---|
| `ENTRY_WINDOW_SEC` | `10` | За сколько секунд до закрытия входить |
| `EXIT_WINDOW_SEC` | `2` | За сколько секунд до закрытия снять все ордера |
| `QUOTE_REFRESH_MS` | `200` | Интервал обновления котировок (мс) |
| `CANCEL_REPLACE_TIMEOUT_MS` | `90` | Бюджет на cancel/replace операцию (мс) |

### Риск

| Параметр | По умолчанию | Описание |
|---|---|---|
| `VOLATILITY_GATE_BPS` | `200` | Блокировать торговлю если диапазон свечи > N bps |

### Статистика и логи

| Параметр | По умолчанию | Описание |
|---|---|---|
| `LOG_LEVEL` | `INFO` | Уровень логирования (`DEBUG` / `INFO` / `WARNING`) |
| `STATS_LOG_INTERVAL` | `10` | Логировать статистику каждые N роллоуверов (~50 мин) |

### Эндпоинты (менять не нужно)

| Параметр | По умолчанию |
|---|---|
| `CLOB_API_URL` | `https://clob.polymarket.com` |
| `BINANCE_WS_URL` | `wss://stream.binance.com:9443/ws` |

## Риски и предостережения

### Финансовые риски

- **Это не гарантия прибыли.** Теоретические расчёты основаны на модели случайного блуждания. Реальные рынки могут вести себя иначе.
- **Ликвидность.** Ордер может не быть исполнен если нет контрагента на Polymarket CLOB.
- **Задержки сети.** Высокий RTT до серверов Polymarket снижает точность cancel/replace. Рекомендуется VPS в US-East или EU.
- **Adverse selection.** Более быстрые боты могут забирать выгодные ордера перед разрешением.

### Технические риски

- **Потеря соединения.** При обрыве Binance WS бот переподключается автоматически (с экспоненциальной задержкой). Во время разрыва открытые ордера остаются в стакане до следующего тика.
- **Аппроксимация результата.** Бот оценивает исход через цену Binance, а не реальный Chainlink оракул. Расхождения возможны при аномальных движениях.
- **Одобрение контрактов.** Если approve не выставлен — ордера будут отклоняться с ошибкой.

### Безопасность ключей

- Храните `POLYMARKET_PRIVATE_KEY` только в `.env` с правами `chmod 600 .env`
- Используйте **отдельный кошелёк** для бота, не основной
- Не держите на кошельке больше средств, чем нужно для торговли
